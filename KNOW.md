# 回帰型ニューラルネットワーク (RNN)
## 学習の流れ
**線形回帰**
  > $x$ : 入力, $y$ : 出力, $x_a$ : 正解データのx, $y_a$ : 正解データのy, $w$ : 重み, $b$ : バイアス
  1. $w$ と $b$ を適当な値で初期化する。
  2. $y = wx + b$ の式に正解データのxを入力し、$y$ を求める。
  3. 求めた $y$ と、$y_a$ に対して損失関数を計算する。
  4. 損失関数を $w$ と $b$ について微分するために、損失関数を逆にたどりながら、$w$ と $b$ についてそれぞれ偏微分していく。
  5. 損失関数の $w$ と $b$ についての導関数にそれぞれ現在の $w$ と $b$ を代入し、勾配を求める。
  6. 現在の $w$ と $b$ から、勾配に学習率を掛けたものを引く(負の勾配: 値を増加させる, 正の勾配: 値を減少させる)。
  7. 2~6を繰り返し、4の勾配をできる限り0に近づける。

## パラメータ
### 損失関数
- **平均二乗誤差** (MSE, L2_Loss): 基本的。外れ値に影響されやすい。
- **平均絶対誤差** (MAE, L1_Loss): 基本的。外れ値に影響されずらい。
- **Huber損失** (Huber_Loss, Smooth_L1_Loss): 誤差が小さい時はMSE、大きい時はMAEのように振る舞う。
- Log-Cosh損失 (Log-Cash_Loss): Huber損失と似ているが、全ての点で二次微分可能。
- 平均二乗対数誤差 (MSLE): 値が指数関数的に変化するターゲット (人口など) に適する。
- 平均絶対パーセンテージ誤差 (MAPE): スケールの異なるデータ間に適する。正解値が0に近いと計算が発散してしまう。
- 分位点損失 (Quantile_Loss, Pinball_Loss): 値ではなく、区間を予測する場合に用いる。

### 最適化手法
- 確率的勾配降下法 (SGD): 基本的。収束に時間がかかる場合がある。
- モーメンタム法 (Momentum): 収束時に振動する動きを抑えることで、SGDよりも早く収束する。
- Nesterovの加速勾配法 (NAG): モーメンタム法の改良版。未来の位置で勾配を計算することで、モーメンタム法よりも早く収束する。
- AdaGrad: 学習率を徐々に小さくする。最終的に学習率が0に近づいて、更新が止まってしまう場合がある。
- RMSProp: AdaGradの改良版。古い勾配情報を忘れることで、学習率が0に近づかないようにする。
- **Adam**: 実質的な標準。モーメンタム法とRMSPropの良いところを組み合わせたもの。
- **AdamW**: Adamの改良版。汎化性能が高い。
- Nadam: AdamにNAGを組み合わせたもの。
- AdaMax: Adamの更新則を無限大ノルムに拡張したもの。学習の安定性が高い。
- AMSGrad: Adamがまれに収束しない問題を解決したもの。
- L-BFGS: 他と違い、二次微分を近似的に計算して用いる。非常に少ないステップで高精度に収束する。全体を一度に計算する必要があるため、巨大なデータには適さない。

### 初期化手法(jax)
- lecun_nomal(): デフォルト値。Sigmoid/Tenh向き。
- he_nomal(): ReLU系の活性化関数向き。
- glorot_uniform(): Xavier初期化。汎用。

### 正規化関数
- シグモイド関数: 基本的な関数。
- ソフトマックス関数: シグモイド関数を多次元に一般化したもの。全体の出力合計が1になる。

# pythonの文法
- pandasのDataFrameをnumpyの配列に変換する際は `.to_numpy()` を用いる。(`.values` は非推奨)
## jax関係
- レイヤー定義には `flax.nnx` を用いる。(`nn` は古い)
- 各データ格納用配列には `jax.Array` 型を用いる。(`jnp.ndarray` はnumpy互換用機能。なお、配列を生成する際は `jnp.array()` を用いる)
